# Product

## Register

product

## Users
[Academic evaluators (professors/students) and hypothetical machine operators/customers. Context is a university full-stack assignment demonstrating IoT integration.]

## Product Purpose
[The platform manages a vending machine network. Users scan QR codes to buy products with internal balance, while admins manage inventory, machines, and sales data.]

## Brand Personality
[Sleek, Friendly, Modern. The interface should feel approachable for consumers and efficient for administrators, moving away from "simple academic" clichés.]

## Anti-references
[Generic Bootstrap admin templates, over-simplified academic project "looks", slow patterns, and cluttered dashboards.]

## Design Principles
- [**Modern Efficiency**: Prioritize fast interactions and clean layouts over legacy admin patterns.]
- [**IoT Transparency**: Make the connection between the physical machine and digital interface feel immediate and reliable.]
- [**Educational Clarity**: While being professional, the architecture (MVC/Service) should be easy to explain during academic evaluation.]

## Accessibility & Inclusion
[WCAG 2.1 Level AA compliance goal, focusing on high contrast for outdoor machine usage and clear navigation for mobile QR code users.]
